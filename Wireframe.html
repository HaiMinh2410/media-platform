Batch Publishing Toast — Stacked Progress + Partial Failure UX
Design a batch publish notification system that tracks per-account publishing status. TOAST CONTAINER: - Fixed position: bottom-right, 28px from edges - Width: 320–380px, dark surface (#161920), 1.5px border (#2a2f42), 12px radius - Entrance animation: translateY(20px)→0 + scale(0.95)→1, cubic-bezier spring, 300ms TOAST HEADER: - Dark surface #1e2230 background - Left: "🚀 Đang xuất bản..." bold 13px - Right: ✕ close button TOAST ITEM LIST — one row per account: Each item row (44px height, 1px bottom border separator): - [30px avatar circle with account color + initial letter] - [account name 12px bold] + [status text 11px muted below] - Right side: status indicator (one of three states): · PENDING: "⏳ Đang chờ..." gray text, no icon · LOADING: spinner (16px circle, 2px border, top arc = accent blue, CSS animation spin 0.8s) · SUCCESS: ✅ emoji icon, status text turns green (#22d3a0) "Thành công" · FAILED: ❌ emoji icon, status text turns red (#ff5c6a) "Thất bại" Items animate through states sequentially (staggered 600ms per account, 1200ms to resolve) TOAST FOOTER (appears only if any account failed): - Dark surface background, flex row space-between - Left: amber badge "⚠️ Một số tài khoản thất bại" (amber text, amber-tinted background pill) - Right: "↻ Thử lại" button (blue text, blue-tinted background pill) - Retry action calls API only for failed accounts (partial retry, not full re-publish) HISTORY CARD (in main list after publish): - Shows post excerpt text (2 lines, ellipsis) - Per-account target chips: green chip "✓ [name]" for success, red chip "✗ [name]" for failed - Date/time meta + success/fail count summary - Status badge top-right: "Thành công" (green) or "Một phần" (amber) - If any failed: "↻ Đăng lại các mục lỗi" link button below chips
