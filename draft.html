// ═══════════════════════════════════════════════ // MASTER MIGRATION PROMPT — Dashboard Phase 1 // From: HTML/CSS vanilla → React + TypeScript + Tailwind v4 // ═══════════════════════════════════════════════ ## Project Overview Migrate và redesign dashboard "InboxAI" quản lý inbox đa nền tảng (Facebook, Instagram). Source: HTML/CSS wireframe đã có. Target: Production-ready React app với design system nhất quán. ## Tech Stack (bắt buộc) - React 19 + TypeScript strict mode - Tailwind CSS v4 - Framer Motion (animations) - Zustand (global state) - cmdk (command bar) - Recharts (nếu cần chart nâng cao) - shadcn/ui (base components) - date-fns + vi locale - lucide-react (icons) - react-countup hoặc custom useCountUp hook ## File Structure src/ ├── components/ │ ├── layout/ │ │ └── TopBar.tsx │ ├── dashboard/ │ │ ├── StatsStrip.tsx │ │ ├── AccountHealthGrid.tsx │ │ ├── AccountCard.tsx │ │ ├── InboxMetricsPanel.tsx │ │ ├── FunnelBar.tsx │ │ ├── LeadTagChart.tsx │ │ ├── AiActivityPanel.tsx │ │ └── AiTile.tsx │ └── ui/ │ ├── Shimmer.tsx │ ├── StatusDot.tsx │ ├── ProgressBar.tsx │ └── PlatformIcon.tsx ├── stores/ │ ├── accountStore.ts │ └── inboxMetricsStore.ts ├── hooks/ │ ├── useCountUp.ts │ └── useInboxMetrics.ts ├── types/ │ └── index.ts └── styles/ └── globals.css ## Design System Font: Sora (sans) + JetBrains Mono (mono) Dark theme chỉ. Base bg: #080810. Accent: Indigo #6366f1, violet #8b5cf6. Semantic: emerald=success, amber=warning, red=error, cyan=info. ## Component Migration Order 1. globals.css + tailwind.config.ts ← design tokens trước 2. types/index.ts ← interfaces 3. stores/accountStore.ts ← Zustand store 4. UI primitives (Shimmer, StatusDot, ProgressBar, PlatformIcon) 5. TopBar (với cmdk integration) 6. StatsStrip (với count-up) 7. AccountCard + AccountHealthGrid 8. InboxMetricsPanel + FunnelBar + LeadTagChart 9. AiActivityPanel + AiTile 10. Dashboard page (compose tất cả) ## Global Rules - KHÔNG dùng any, đặt types rõ ràng hết - Tất cả animation qua Framer Motion, không CSS inline animation (ngoại trừ shimmer có thể dùng CSS @keyframes vì performance) - Mỗi component: props interface rõ + JSDoc comment tóm tắt - Mock data trong file riêng: src/data/mockDashboard.ts - Error states và loading states phải handle hết - Responsive: mobile-first, breakpoints sm/md/lg/xl ## Animations phải có - Count-up: StatsStrip + AiTile values (mount) - Stagger reveal: AccountCards (0.05s delay mỗi card) - Progress bars: animate width 0→value (0.6s ease-out) - Status dot pulse: chỉ error state (CSS keyframes) - Shimmer: AiDraftShimmer bars - Command bar modal: scale 0.96→1 + opacity 0→1 (0.15s) - Funnel bars: animate width khi data thay đổi (activeRange change) ## Bắt đầu từ đâu? Hãy build theo thứ tự trong "Component Migration Order" ở trên. Bắt đầu với: globals.css + types/index.ts + stores/accountStore.ts Sau đó tiếp tục từng bước một, mỗi bước confirm trước khi sang bước tiếp.
