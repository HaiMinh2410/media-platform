// section 04
Architecture & API
Luồng dữ liệu từ backend → React state → components. Tech stack đề xuất và endpoint structure.
🔗 DATA FLOW — Dashboard state architecture
[ Backend API / Webhook Server ]
↓ REST polling (30s) + optional WebSocket
[ React Query / SWR (data fetching layer) ]
├── useAccountsQuery() → GET /api/accounts
├── useStatsQuery() → GET /api/stats/overview
├── useInboxMetrics() → GET /api/inbox/metrics?account=&period=
└── useAISummary() → GET /api/ai/summary
[ Dashboard.tsx (root component) ]
├── <StatsStrip /> (stats data)
├── <AccountHealthGrid /> (accounts array)
└── <TwoColumnLayout>
├── <InboxMetricsPanel />
└── <AISummaryPanel />
A
Polling strategy: Stats + AI Summary poll mỗi 30s. Account status poll mỗi 60s. Inbox metrics chỉ fetch khi user thay đổi filter.
B
Error boundary: mỗi panel bọc trong ErrorBoundary. Nếu 1 API fail → chỉ panel đó show error state, không crash toàn dashboard.
C
WebSocket events: khi có tin nhắn mới, server push event → invalidate useStatsQuery để re-fetch.
